
## 🚀 실행 순서

1. **사전 요구 충족**: 로컬에서 Ollama가 실행 중이어야 하며, 필요한 파이썬 패키지(`chromadb`, `telebot` 등)가 설치되어야 합니다.
2. **음악 파일 색인화 (Indexing)**:
   ```bash
   python app/indexer.py
   ```
   - 이 스크립트는 로컬 디렉토리의 음악 파일을 읽어 메타데이터를 추출하고 AI 임베딩을 생성하여 ChromaDB에 저장합니다.
3. **텔레그램 봇 실행 (Running Bot)**:
   ```bash
   python app/main.py
   ```
   - 봇이 대기 상태가 되며, 텔레그램에서 명령어 및 자연어 검색 요청을 수신할 준비를 마칩니다.

## 🎛️ 사용 방법

1. 텔레그램에서 봇과 1:1 대화를 시작하거나, 봇을 운영 중인 음악 채널에 관리자로 초대합니다.
2. 다음과 같이 자연어로 원하는 음악을 요청합니다:
   - "기분 좋은 팝송 3개만 추천해줘"
   - "BPM 빠른 아이유 노래"
   - "90년대 잔잔한 노래 틀어줘"
3. 봇이 사용자의 요청을 분석하여 가장 적합한 음악 파일을 텔레그램 채널(또는 대화방)으로 전송합니다.

## ⚙️ 어떻게 동작하는지 (작동 원리)

- **`app/indexer.py` (색인 단계)**:
  1. 로컬의 음악 파일들을 순회하며 메타데이터(제목, 가수, 장르, BPM 등)를 추출합니다.
  2. Ollama를 활용해 해당 음악 정보를 텍스트로 요약하고 벡터(Embedding)로 변환합니다.
  3. 벡터값과 메타데이터를 로컬 벡터 데이터베이스인 ChromaDB에 저장합니다.

- **`app/main.py` (검색 및 출력 단계)**:
  1. 텔레그램 봇이 사용자의 메시지를 수신합니다.
  2. Ollama AI 모델이 사용자의 자연어 질문을 분석하여 의도를 파악하고, 이를 검색에 적합한 형태로 처리합니다.
  3. 변환된 쿼리 벡터를 이용해 ChromaDB에서 의미적으로 가장 유사한 음악을 검색합니다.
  4. 검색된 음악 파일의 로컬 경로를 찾아 텔레그램 채널에 오디오 파일로 전송합니다.

## 🔮 향후 개선점

- **고급 오디오 분석**: 단순 메타데이터 의존을 넘고, `librosa` 등의 라이브러리를 활용해 실제 오디오 피처(무드, 비트 패턴 등)를 직접 분석하여 벡터에 추가.
- **상호작용 강화**: 텔레그램 인라인 키보드(Inline Keyboard)를 도입하여 여러 검색 결과 중 원하는 곡을 선택할 수 있는 페이지네이션 기능 구현.
- **사용자 피드백 루프**: 사용자의 '좋아요/싫어요' 반응을 수집하여 향후 유사도 검색 가중치를 조정하는 개인화 추천 기능 추가.
- **스트리밍 및 외부 연동**: 로컬 파일뿐만 아니라 유튜브 링크나 스트리밍 서비스(Spotify 등) 검색 결과를 함께 제공하는 기능 확장.
